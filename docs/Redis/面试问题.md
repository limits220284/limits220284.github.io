# 面试问题

## Redis为什么快？

- **基于内存存储；**
- **单线程模型，少了多线程的切换开销；**
- **高效的数据结构**，对于字符串的操作基本都是O(1)的，一些操作肯定也是经过优化处理的；
- **基于事件驱动的异步I/O**:
  - Redis使用事件驱动模型，采用非阻塞I/O，通过epoll等机制实现多个客户端的高效处理；Redis通过事件循环来处理各种请求，减少了线程切换和同步操作的开销，提高了系统的并发性能；
- **优秀的持久化机制**：可以基于异步方式持久化，不阻塞主线程；

## Redis内存满了应该怎么办？

- 配置合理的内存限制：redis.conf中，可以设置maxmemory参数来限制Redis可使用的最大内存；
- 内存淘汰策略，LRU，LFU，随机淘汰策略；
- 增加服务器内存，然后调大redis内存；
- 开发层面，设置均匀的过期时间；
